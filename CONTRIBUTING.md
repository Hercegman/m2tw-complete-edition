# Contributing

Contributions are welcome! Fork the repo, make your change on a branch, and
open a **pull request** — the maintainer reviews and merges.

```
1. Fork this repo on GitHub (button top right)
2. git clone https://github.com/<you>/m2tw-complete-edition
3. git checkout -b my-change
4. ...work...
5. git push origin my-change  ->  open a Pull Request against main
```

## IMPORTANT: generated vs. hand-edited files

A large part of `mod/complete_edition/data/` is **generated** by the Python
tools in `tools/`. Do NOT hand-edit generated files — change the generator
(or its source data) instead, otherwise the next build overwrites your work.

| Generated by | Files |
|---|---|
| `tools/add_faction_data.py` | export_descr_unit.txt, export_descr_buildings.txt, descr_names.txt, descr_character.txt, descr_banners_new.xml, text/*.strings.bin, menu symbol copies |
| `tools/fix_strat_positions.py` | character coordinates in descr_strat.txt |
| `tools/gen_model_battle.py` | descr_model_battle.txt, descr_caps_ex.txt |
| `tools/gen_heraldry.py` | Balkan faction icons/banners/symbols (composited into CA canvases) |
| `tools/gen_standards.py` | descr_standards.txt, banners/symbols9-11.tga, standard_index values |
| `tools/gen_unit_cards.py` | ui/units/<faction>/, ui/unit_info/<faction>/ |
| `tools/gen_logos.py` | ui/*.sd.xml, stratpage_ce/sharedpage_ce atlases, logo_index values |
| `tools/extend_faction_select.py` | data/menu/mtw2.lnt |

Hand-edited (safe to change directly): `descr_sm_factions.txt` (faction
blocks; generators only patch specific fields), `descr_strat.txt` (except
character x/y), `descr_regions.txt`, `docs/`, the generators themselves.

## Build & validate

Requires Python 3 and the reference extracts (see below).

```
python3 tools/add_faction_data.py
python3 tools/fix_strat_positions.py
python3 tools/gen_model_battle.py
python3 tools/gen_heraldry.py
python3 tools/gen_standards.py
python3 tools/gen_unit_cards.py
python3 tools/gen_logos.py
python3 tools/extend_faction_select.py
python3 tools/validate_faction_order.py   # must print OK
python3 tools/validate_strat.py           # must print OK
```

`./deploy.sh` copies the mod into your game folder (edit the GAME path at the
top if yours differs) and runs the validators as a gate.

Some generators need reference data that is NOT in the repo (too big /
game-derived): extracts of the game packs and Kingdoms DLC packs under
`research/dlc-extract/` and `research/base-extract/data_cards/`. Use the
M2EX pack tool to produce them: `tools/pack.exe x <pack> <outdir>` (pack.exe
ships with M2EX in `<game>/tools/`). See `docs/m2ex-notes.md`.

## PR guidelines

- One topic per PR; explain WHAT you changed and WHY.
- Run both validators before pushing — a PR that fails them will not merge.
- Commit messages in English.
- New factions must go through the full pipeline (see `docs/faction-map.md`
  for the roster plan and per-faction checklist).
- Test in game where possible and mention what you tested (campaign start,
  recruitment, battles, save/load).
